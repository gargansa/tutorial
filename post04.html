Pipes, Redirects, and Tees

Useful powerful commands that simplify code. Mouse over bold text for explanation of command.

Chaining Commands

Different ways to run commands.

Using a semicolon to separate commands

Run commandA then commandB

commandA; commandB

grep -l 'hello' file.txt; echo "hello" > file.txt

Using double ampersand to separate commands.

If commandA is successful then run commandB

commandA && commandB

grep -l 'world' file.txt && echo "world" > file.txt

Using double pipe to separate commands.

If commandA fails then run commandB

commandA || commandB

grep -l 'world' file.txt || echo "world" > file.txt

Using a single ampersand to run a command in the background.

command &

tail file.txt &

Pipe

Sending the results of a command to a new command.

grep "string" filename.txt | grep "a"

Redirects

Sending the results of a command to a file.

echo 'Hello World' > filename.txt

echo 'Hello World' >> filename.txt

Tee

Splitting the results of a command between a file and its standard output.

echo "hello world" | tee filename.txt

echo "hello world" | tee -a filename.txt
